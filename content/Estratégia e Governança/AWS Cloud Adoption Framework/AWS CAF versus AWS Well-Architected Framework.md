AWS CAF e AWS Well-Architected Framework tratam de problemas diferentes, embora os dois ajudem organizações que usam nuvem.

Os dois frameworks são da AWS, mas respondem perguntas diferentes.


Pense assim:


* [[AWS Cloud Adoption Framework (AWS CAF)|AWS CAF]]: a organização está pronta para adotar nuvem?
* [[AWS Well-Architected Framework]]: este workload está bem arquitetado?


CAF olha transformação organizacional.

Well-Architected olha qualidade de arquitetura.

---

## AWS CAF

Use quando o enunciado falar sobre:

* adoção de nuvem;
* transformação digital;
* prontidão;
* pessoas;
* governança;
* estratégia;
* roadmap;
* capacidades organizacionais;
* perspectivas Business, People, Governance, Platform, Security e Operations.

---

## AWS Well-Architected Framework

Use quando o enunciado falar sobre:

* revisar workloads;
* boas práticas arquiteturais;
* pilares;
* confiabilidade;
* segurança;
* eficiência de performance;
* otimização de custos;
* excelência operacional;
* sustentabilidade.

---

## Como Diferenciar

Um cenário pode tentar confundir dizendo que uma empresa quer “melhorar a adoção da nuvem” e colocar Well-Architected como alternativa.

Se o foco é adoção organizacional, a resposta é [[AWS Cloud Adoption Framework (AWS CAF)]].

Se o foco é revisar uma arquitetura de aplicação, a resposta é [[AWS Well-Architected Framework]].

---

## Ideia Central

* [[AWS Cloud Adoption Framework (AWS CAF)|CAF]]: transformação da organização.
* [[AWS Well-Architected Framework|Well-Architected]]: qualidade do workload.


---

## Outra forma de entender

Uma boa resposta precisa ter três partes:


* definição: o que é.
* função: para que serve na adoção da nuvem.
* diferença: com o que não deve ser confundido.


No contexto do AWS CAF, o objetivo não é decorar nomes. O objetivo é reconhecer qual área da organização precisa amadurecer.

Quando o cenário descreve um problema, identifique se ele é de negócio, pessoas, governança, plataforma, segurança ou operações. Essa leitura normalmente entrega a resposta.
